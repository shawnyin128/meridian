Meridian Full Architecture

两个产品，一个研究系统：Paper Wiki 是 workflow 产品；Lab 是 idea graph product。 Paper Wiki 可以作为 MCP delivery，Lab 和外部普通 coding workflow 通过稳定工具接口使用 wiki，而不是耦合 vault 内部结构。

Boundary

Wiki 维护 canonical research memory；MCP 暴露 context/search/read/write-back；Lab 管理 idea graph；外部普通 coding workflow 负责代码实现、诊断路径和测试。

Wiki-only

只读 paper、Zotero 批注、维护 claim/method/idea，不进入代码仓库。

Lab-only

只在代码仓库里管理 idea graph、实验设计、结果解释和实验记忆。

Integrated

Lab 先向 Wiki 要 context packet；外部 coding workflow 返回实验结果后，用 write-back packet 更新研究记忆。

Implementation Order

先构建 Paper Wiki。Lab 之后再接入，因为它的 wiki-grounded idea graph 能力依赖一个真实可用的 wiki backend。

1. Brainstorm

界定 prototype 边界、非目标、人工审查点和评测维度。

2. Prototype

做最小 Paper Wiki：ingest、user insight、context packet、draft/publish。

3. Human Review

人工检查是否真的符合 paper reading 和 idea refinement 需求。

4. Test Cases

大量案例写清问题和预期结果，但允许多条有效路径。

5. Evaluation

评估 artifact quality、provenance、retrieval、write boundary、failure honesty。

6. Refine

失败归因到 workflow、schema、retrieval、MCP/tool interface，再迭代。

Source-Grounded Principles

来自 LLM Wiki 指南、Anthropic agent engineering posts 和 followup 经验：原则进入系统边界，具体社区实现不照搬。

Simple First

固定任务用 workflow；idea graph 用轻量 Lab；自适应代码执行交给外部普通 coding workflow。

Progressive Disclosure

先 metadata / index / packet，再按需展开 page、source、tool schema。

High-Signal Context

返回选择原因、provenance、conflicts、gaps，而不是 retrieval dump。

Agent-Ergonomic Tools

MCP tools 要少、清晰、可评测，支持 concise/detailed 输出。

Reviewable Writes

先 propose/apply，再改 canonical wiki；判断点显式暴露。

Persistent Learning

把 query、失败实验、设计决策、stale claims 都沉淀回 wiki。

Impact Rollback

科研代码用 staged commits 保留可比较、可回退、可归因的时间轴。

Paper Wiki MCP Delivery

Paper Wiki 的本体仍是 raw sources + Markdown wiki + schema。MCP 是 delivery boundary，让 Lab 和外部普通 coding workflow 通过稳定工具契约使用研究记忆。

Stable Boundary

客户端不需要知道 paper/claim/method 页面实际放在哪里，只请求 context、page、search、write-back。

Search

查 papers、claims、methods、ideas、experiments、results，并返回 provenance。

Context Packet

为科研问题或 coding task 选择小而准的 pages、证据、冲突和缺口。

Read Page

读取具体 wiki page、source summary、frontmatter 和引用关系。

Propose Write-back

接收实验/代码证据，生成可 review 的 wiki 更新建议，而不是静默改 canonical memory。

Lab Boundary

Lab 产品最低成立条件有三个：维护 idea graph continuity；作为 Paper Wiki 的 idea-side gateway；把代码实现、调试和测试交给外部普通 coding workflow。

1. Idea Graph Continuity

维护 threads、approach nodes、experiment evidence 和 local finding proposals。

2. Wiki Gateway

在 feasibility、placement、handoff 和结果解释中主动查 paper/wiki memory。

3. Development Handoff

把 implementation/debug/test 工作交给外部普通 coding workflow，并保留 evidence identity 和 return criteria。

Target Artifact

Lab state 要显式保留 idea placement、approach status、experiment evidence、finding proposal 和 handoff packet。

External Coding Workflow

自适应读代码、跑命令、加诊断、查日志、决定 sanity check 或 ablation 属于普通 coding workflow。

Wiki Awareness

遇到 paper definition、method detail、prior failed path、metric claim、user insight 时主动请求 context packet。

Thin Guardrails

不套重流程；只要求 learning target、evidence identity、handoff packet、write-back packet。

Paper Wiki Workflow

预定义 workflow。目标是稳定维护研究状态，并通过 MCP deliver 给不同客户端。

workflow product

Ingest Paper

raw paper、Zotero annotations、user insight 进入 draft analysis。

Evolve Analysis

基于反馈更新 paper model、claims、methods、open questions。

Retrieve Context

返回小而准的 context packet，而不是一堆搜索结果。

Maintain Memory

更新 index、log、graph、paper/claim/method/idea/result pages。

Context Packet

Paper Wiki MCP -> Lab / external coding workflow: 相关 pages、选择原因、证据、冲突、缺口、允许写入策略。

knowledge grounding

Write-back Packet

Lab / external coding workflow -> Paper Wiki MCP: 实验目的、命令/config、结果、解释、claim 影响、失败路线、下一步。

Lab Idea Graph

Idea graph product。目标是把自由探索转成 researcher 看得懂、可维护、可回写的 idea state 和证据。

idea graph product

Place Idea

确认 root、child、sibling 或 link，并保留 raw idea 与 normalized hypothesis。

Ground With Wiki

查 wiki context，识别 support、contradiction、uncertainty 和 missing evidence。

Prepare Handoff

当需要代码实现、调试或测试时，生成给外部普通 coding workflow 的 handoff packet。

Interpret And Next

判断结果是否支持 hypothesis，是否需要 control、ablation 或停止方向。

Shared Research Memory

两件产品共享同一套研究状态，但拥有不同控制模型。Wiki 负责 canonical memory；Lab 负责把 idea graph 和实验过程变成可回写的证据。

Papers

论文理解、批注、隐藏实现细节。

Claims

支持、反驳、置信度、证据来源。

Methods

公式、实现、probe、ablation 关系。

Experiments

config、command、metric、run evidence。

Ideas

hypothesis、failed paths、next decisions。

Paper-grounded Ideas

想法不是凭空生成，而是从 paper、claim、method、failed path 中被 refine。

Experiment-ready Code Work

代码允许为 ablation/probe 保留冗余，但必须服务明确观察目标。

Persistent Research Learning

每次结果都回到 wiki：说明什么、不说明什么、下一步是什么。

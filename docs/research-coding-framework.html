Research Coding Loop

一个轻量科研写码框架：用 LLM Wiki 保存研究状态，用 Arbor 的 evidence/checkpoint 思想保持续航， 只在真正开放、可并行的研究问题上使用 multi-agent burst。

Boundary

不是 Arbor for science，也不是默认多 agent 编码。核心是一条 paper、idea、code、experiment、result 互相更新的 workflow。

Research Wiki State

papers、claims、methods、ideas、hypotheses、experiments、results、open questions。保存研究理解。

Code Work State

repo map、active slice、changed files、command plan、test plan、environment constraints。约束写码范围。

Experiment Result State

configs、commands、logs、metrics、plots、failed runs、interpretation。把运行结果变成证据。

Experiment Design

把 hypothesis 拆成最小可跑实验、controls、ablation 和 stop condition。

Sanity Check

先证明训练 loop、metric、data split、config 开关没有明显坏掉。

Result Interpretation

把 logs、metrics、plots、失败原因转成下一步 research decision。

Experiment Memory

记录跑过什么、为什么跑、说明什么、还缺什么，避免跑过但忘了。

Reproduction Diagnosis

定位 paper、repo、data、hyperparam、hardware、metric 之间哪里对不上。

Main Loop

每一步都产出可落盘状态；失败实验也必须回写为研究记忆。

workflow first
1

Classify Event

判断是实验设计、sanity check、结果解释、实验记忆、复现诊断，还是普通 coding。

2

Retrieve Context

从 paper wiki、claim/method、旧 idea、失败路线、实验结果和代码入口生成 context packet。

3

Select Workflow

只选一个主链路，避免把一次请求膨胀成设计、实现、debug、记录全包。

4

Plan Slice

定义最小动作：research question、expected learning、文件/命令、证据和停止条件。

5

Run Or Diagnose

执行最小代码/实验/诊断动作，并保留 config、command、metric、output 和失败信息。

6

Interpret And Update

解释结果，更新 wiki、lab log、hypothesis、failed-path memory 和下一步。

Keep From Arbor

  • project guide and short-term memory
  • decision trace and evidence artifacts
  • done-when, adapted to research learning
  • outcome-first evaluation and resume discipline

Use Multi-Agent Only For Research Burst

  • literature breadth and method search
  • benchmark or dataset landscape
  • independent claim/result review
  • not for tightly coupled code edits

Context Packet

少量真正相关的 paper、claim、method、code entrypoint 和 gaps。

Research-Code Slice

一个最小可验证的实验或代码动作，带学习目标和停止条件。

Updated Research Memory

wiki、result、hypothesis、failed path 和下一步 idea 都被持续更新。

Meridian LLM Wiki Architecture

一个 Markdown-first 的 research state compiler：原始材料保持不可变，LLM 把它们持续编译进 Obsidian 友好的 wiki， schema 和日志保证每次维护都可审查、可追踪、可继续。

Raw Inputs Agent Work Compiled Wiki Control Plane Human Surface
1. Input Channels

科研信号从多个工具进入，但不直接成为最终状态。

Conversations

GPT、Claude、Codex 讨论里的 idea、判断、路线选择、next action。

Zotero / Reading Notes

paper critique、疑问、摘要、人工洞见、Better BibTeX 或 Markdown export。

Papers / Web / NotebookLM

论文、网页剪藏、NotebookLM grounded analysis、外部报告。

Experiments

实验计划、代码 repo、日志、图表、失败记录、结果解释。

2. Integration Workspace

把输入变成可审查的 wiki 更新，而不是一次性回答。

Capture Inbox

低摩擦收集材料，先分类为 source、idea、claim、project、decision 或 synthesis 候选。

normalize and classify

Integration Agent

识别新增对象、更新旧页面、合并重复、建立双链、标记冲突和证据缺口。

produce reviewable diff

Provenance Guard

区分 source fact、wiki synthesis、user decision；claim 必须尽量指回来源。

3. Compiled Markdown Wiki

唯一长期可信的研究状态层。LLM 维护结构，人类在 Obsidian 中阅读、导航和判断。

Research Objects

Idea、Paper、Claim、Concept、Project、Decision、Synthesis。每个页面都有 frontmatter、状态、来源、链接和下一步。

Raw Source References

`raw/` 保持不可变；wiki 中的 source page 解释材料价值、范围和引用位置。

Evidence Graph

supports、contradicts、related、derived-from 等关系通过 Markdown 双链和 frontmatter 表达。

Index

`wiki/index.md` 是内容目录，agent 查询时先读它，再用 `rg` 定位具体页面。

Log

`wiki/log.md` 记录 ingest、query、lint、plan 的时间线，append-only 且可用简单命令解析。

Templates

页面模板固定最小 frontmatter，避免 wiki 演化成不可维护的自由文本集合。

Dataview / Graph View

Obsidian 负责浏览、图谱、动态表格；Meridian 不急着重做这些界面能力。

4. Control and Outputs

控制面让 agent 有纪律；输出面让研究继续推进。

Schema Control

`AGENTS.md`、`SKILL.md`、templates、frontmatter 规则定义 agent 如何写入。

Maintenance Policies

diff-first、source-readonly、log append-only、claim provenance、human judgment boundary。

Human Review Surface

Obsidian、git diff、graph view、Dataview dashboard、Markdown pages。

Derived Outputs

answers、comparison tables、research plans、experiment plans、Marp slides、charts、canvas。

Lifecycle Operations

C

Capture

把材料放进 raw 或 inbox；只做轻量分类，不急着过度抽象。

I

Ingest

一次处理一个 source，生成 source page，并更新相关对象页、index 和 log。

Q

Query

先查 wiki，而不是重新读全部 raw source；有价值的 synthesis 反写入 wiki。

L

Lint

找冲突、孤页、缺证据 claim、停滞 project、未整合 inbox。

P

Plan

从当前研究状态推出下一轮阅读、实验、暂停条件或决策更新。

Architecture rule: Meridian 不是文档上传问答工具。它的核心是把每次阅读、讨论和实验观察编译成可审查、可链接、可持续维护的 Markdown 研究状态。

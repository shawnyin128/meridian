Meridian Paper Wiki Entry Architecture

最终入口只有两个：Prompt/Skill 和 MCP。每个入口只服务两个场景：Update Wiki 和 Use Wiki。CLI 是底层执行原语。

User Intent 研究、写码、读 paper、记录 insight、沉淀 synthesis
→
Prompt / Skill Entry meridian-paper-wiki 总入口 skill
MCP Entry context, read, trace, update, propose, apply
→
Update Wiki ingest, insight, write-back, refine, audit
Use Wiki retrieve, read, trace, answer
→
Meridian Core source management, canonical publish, retrieval v1, proposals, evolution, audits
Canonical Vault papers, methods, topics, concepts, claims, evidence, syntheses, index/log/catalog
Update Wiki Output managed source, canonical page, user insight, proposal, synthesis, refinement, audit report
Use Wiki Output compact context packet, selected page sections, provenance chain, answer-ready evidence
Source of Truth wiki/ Markdown vault
Internal .drafts/, .versions/, judge/debug artifacts
Write Policy proposal → lint → publish
CLI testable execution primitives

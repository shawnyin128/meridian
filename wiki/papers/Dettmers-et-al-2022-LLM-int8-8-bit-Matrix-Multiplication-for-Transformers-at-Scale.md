## Mechanism

### Vector-wise int8 quantization

- Purpose: Uses row-wise activation scales and column-wise weight scales for int8 matrix multiplication so transformer linear layers can run in 8-bit with limited error.
- Operates on: activation matrix; weight matrix; row/column absmax scales.
- Produces: int8 operands; int32 matmul result; dequantized output.
- Depends on: vector-wise scaling is enough for non-outlier dimensions; hardware kernels can exploit int8 matrix multiplication efficiently.
- First checks: Unit test the scale/dequantization path against FP16 matmul on non-outlier tensors; Track row/column scale shapes explicitly to avoid silent broadcasting errors.
- Source: Provenance: p. 7, p. 1 / Abstract, p. 2.

### Mixed-precision decomposition for outlier features

- Purpose: Separates sparse high-magnitude outlier feature dimensions into a higher-precision path while using int8 multiplication for the dense majority.
- Operates on: outlier feature threshold; hidden states; projection weights.
- Produces: int8 dense path; FP16 outlier path; combined output.
- Depends on: outlier dimensions are sparse enough that the FP16 path is cheap; the selected threshold preserves model quality across scale.
- First checks: Ablate the outlier threshold and record both perplexity and latency; Keep the dense and outlier matmul outputs separately inspectable before summation.
- Source: Provenance: p. 7, p. 1 / Abstract, p. 2.
